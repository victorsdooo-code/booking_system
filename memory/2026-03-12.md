# 2026-03-12 - Dev1 Memory

## Sprint 2 Scheduler UI

### Technical Notes & UI Decisions

**File Structure:**
- `admin.html` - Admin panel with new 排班管理 (Schedule Management) section
- `index.html` - Booking page with API-driven time slot selection

**UI Design Decisions:**
1. **Sidebar Navigation**: Added "📋 排班管理" button between "新增預約" and "登出"
2. **Form Layout**: Reused existing `.filters-container` and `.filters-grid` CSS classes for consistency
3. **Table Design**: Used existing `.table-container` styling with 9 columns
4. **Day of Week**: Stored as 0-6 (0=Sunday, 1=Monday, etc.) for database compatibility
5. **Time Format**: HTML5 time input (HH:MM format)

**API Endpoints Integrated:**
- `GET /api/admin/schedules?token=` - Fetch all schedules
- `POST /api/admin/schedules?token=` - Create schedule
- `PUT /api/admin/schedules/:id?token=` - Update schedule
- `DELETE /api/admin/schedules/:id?token=` - Delete schedule
- `GET /api/slots/available?doctorId=&date=` - Get available slots for booking

**Schedule Data Structure:**
```javascript
{
  id: number,
  doctorId: number (1=中醫師，2=物理治療師),
  serviceId: number (1-4),
  dayOfWeek: number (0-6, 0=Sunday),
  startTime: string (HH:MM),
  endTime: string (HH:MM),
  slotDuration: number (minutes: 15/30/45/60),
  maxAppointments: number
}
```

**Booking Page Changes:**
- `populateTimeSlots()` now async function
- Calls API before rendering slots
- Shows loading state during fetch
- Displays error message if API fails
- Falls back to default 9:00-18:00 slots on error
- Marks unavailable slots as disabled (greyed out)

**Error Handling:**
- Separate error log section for schedules (`#scheduleErrorLogs`)
- Console logging for debugging
- User-friendly alerts for save/delete operations
- Fallback behavior for booking page if slots API unavailable

**State Management:**
- `allSchedules` - Array of all schedules
- `editingScheduleId` - Tracks which schedule is being edited (null for new)

**Helper Functions Added:**
- `getDoctorName(doctorId)` - Map ID to Chinese name
- `getServiceName(serviceId)` - Map ID to service name
- `logScheduleError(message)` - Log to schedule error panel
- `generateDefaultTimeSlots(container)` - Fallback slot generator
