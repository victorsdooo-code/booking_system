### API Endpoints Used:

**Public (Booking):**
- GET `/api/clinics`
- GET `/api/doctors`
- GET `/api/services`
- GET `/api/schedules?doctorId=&date=`
- POST `/api/appointments`

**Admin (requires X-Admin-Token header):**
- GET `/admin/appointments?search=&sort=`
- GET `/admin/clinics`
- POST `/admin/clinics`
- PUT `/admin/clinics/:id`
- DELETE `/admin/clinics/:id`
- POST `/admin/schedules/batch-copy`

### Key Decisions:
1. **Auth Method:** Changed admin auth from query param `?token=` to header `X-Admin-Token` (more secure)
2. **Error Handling:** All API calls wrapped in try-catch with user-friendly messages
3. **Fallback Strategy:** Client-side mock data as fallback if API fails
4. **Search/Sort:** Hybrid approach - try server-side first, fallback to client-side
